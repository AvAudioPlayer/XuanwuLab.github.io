Tencent Xlab安全公告披露原则

作者： admin   2015-12-03

    Any intresting software, platform and device are our targets, Xlab often discovers new bugs and vulnerabilities in those targets. To manage those bugs and vulnerabilities each one will be assigned a xlab id. A security advisory without details will be created when a vulnerability is found. To make the world safer, Xlab will happlily collaborate vulnerable vendor to fix those bugs and vulnerabilities. Once a patch is available, after a while security advisory contains details becomes public.

    When in such cases, Xlab can decide the date to release detailed security advisory.

  1. Vulnerable vendor is nonresponsive, Xlab has made the best efforts to work with the vendor.

  2. Vulnerability information is already publicly available, or vulnerability is exploited in the
    wild. Other security researcher finds the same vulnerability and make it public.

  3. Patch is very difficult to build.

The mount of details in security advisory depends on the criticality and impact of
the vulnerability.
